description: Start a new feature development
---
1. Understand the user's requirements for the new feature.
2. If necessary, use `@brainstorming` to generate ideas.
3. Once the plan is clear, use `@implementation-plan` to create a detailed plan.
4. Implement the feature following the plan.
5. Verify the implementation with tests.
// turbo
6. Commit the changes using the project's commit convention.
